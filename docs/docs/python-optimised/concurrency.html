Concurrent Processing

What to Expect in This Chapter

Certain problems naturally lend themselves to a concurrent or parallel processing strategy. A classic example is a Monte Carlo (MC) simulation of particle histories. There are several ways to implement concurrency in Python. In this chapter, I will show you how to do this effectively—and what pitfalls to avoid.

1 Making Sense of Concurrency

Let us begin with a simple analogy. Imagine three people (A, B, C) at a supermarket checkout.

  • Synchronous Processing: They are queuing at a single teller, who processes them one at a time, in sequence. This is an example of a synchronous or serial process. The total time taken is:

    \[ t_{\text{total}} = t_A + t_B + t_C \]

  • Multiprocessing: A faster approach is to have multiple tellers. A, B, and C are then processed in parallel, independently of one another. In this case, the total time is determined by the slowest of the three: \[ t_{\text{total}} = \max(t_A, t_B, t_C) \]

    This is an example of multiprocessing. Since modern computers have multiple cores, this approach is relatively straightforward to implement in Python.

  • Asynchronous Processing: Now lets consider a different way to speed things up. Suppose there is only one teller, who is currently helping A. While A is searching for his credit card, the teller temporarily starts serving B. If B drops their coins, the teller might switch back to A, or even begin helping C. This is an example of asynchronous processing. Notice how the switching is event-driven. \[ t_{\text{total}} < t_A + t_B + t_C \]

  • Multithreading: A subtle variation of the above approach is to allow the teller to switch between customers whenever the teller decides to. This is called pre-emptive switching, and this form of asynchronous processing is known as threading.
    In fact, this is how most operating systems manage multitasking: the system switches between tasks so rapidly that it appears many things are happening at once.

1.1 A fly in the ointment

Python uses a mechanism called the Global Interpreter Lock (GIL) to manage memory safely and efficiently. However, the GIL prevents multiple threads from running truly in parallel, which limits the effectiveness of threading in Python, especially for CPU-bound (i.e. processes that require intensive use of the CPU) tasks. As a result, asynchronous programming (using asyncio) or multiprocessing is often a better choice when you need concurrency.

2 Implementing Concurrency in Python

Let me now show you how to implement these concurrent methods. Lets start with a simple task can use to compare these methods.

def task(name, duration):
    duration = duration
    print(f"{name} started, will take {duration:.2f}s")
    time.sleep(duration)
    print(f"{name} finished")

import time
import asyncio

async def async_task(name, duration):
    duration = duration
    print(f"{name} started, will take {duration:.2f}s")
    await asyncio.sleep(duration)
    print(f"{name} finished")

async def run_asyncio():
    print("\n--- AsyncIO Execution ---")
    start = time.time()
    async_tasks = [async_task(name, duration) for name, duration in task_info]
    await asyncio.gather(*async_tasks)
    print(f"Total time: {time.time() - start:.2f}s")
